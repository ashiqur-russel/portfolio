## Why Separate Workflows?

### Advantages:

1. **Clarity**
   - `ci.yml` = Code Quality
   - `cd.yml` = Deployment
   - Easy to understand what each does

2. **Independence**
   - Can modify CI checks without touching deployment
   - Can change deployment process without affecting CI

3. **Better Logs**
   - Separate workflow runs in GitHub Actions
   - Easier to debug issues
   - Clear success/failure indicators

4. **Flexibility**
   - CI can run on all branches
   - CD only runs on specific branches
   - Can add more workflows later (e.g., nightly tests)

5. **Security**
   - CD workflow has deployment secrets
   - CI workflow doesn't need those secrets
   - Better separation of permissions

## Summary

- **Feature branches**: CI runs (no deployment)
- **Development branch**: CI + CD to staging
- **Main branch**: CI + CD to production
- **All PRs**: CI must pass before merge
- **Deployment**: Only happens after CI passes
